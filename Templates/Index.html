TYPO3.Fluid Rendering Engine

Fluid is a modern templating engine based on a xml based syntax. It helps you create clean and functional templates while maintaining performance. Templates can be compiled into static php cache files, that are amazingly fast. It is based on PSR-4, requires no dependencies, is easy to integrate and can be extended in various places.

Easy to write

The syntax is based on html and let's you transform your design into code with ease

Ludicrous speed.

Fluid compiles to a pure php cache that enables speed, you don't have to worry about.

Built to be flexible and extandable

From adding custom viewHelpers to replacing core classes it's built to be flex to your needs.

Basic Syntax

Fluid has two main syntax formats - the tag mode and the inline/shorthand mode.

{items}

Is the same as:

{items -> f:count()}

The reason for this is, so that you can cleanly nest viewHelpers like this:

ViewHelpers

ViewHelpers are fluids concept of enabling logic inside by mapping xml tags to viewHelper classes.

In order to use ViewHelpers you need to register a namespace for it, in this case this could look like:

$view->getViewHelperResolver()->registerNamespace('my', 'Foo\My\ViewHelpers');

Then it outputs "Hey John, how are you?" because fluid can parse this and map it's arguments to the ViewHelper class like this:

class HelloWorldViewHelper extends AbstractViewHelper { /** * @return void */ public function initializeArguments() { parent::initializeArguments(); $this->registerArgument('name', 'string', 'Tell me your name', TRUE); } /** * Outputs a nice greeting based on a name * * @return string */ public function render() { return sprintf('Hey %s, how are you?', $this->arguments['name']); } }

You can define any number of arguments, required, optional, type restricted, etc. To find out more check: ViewHelpers - what these classes do in the Fluid language
